# Valley — cross-platform voxel world

A streaming voxel world that runs from **one `.cljc` codebase** on two backends:

- **JVM** → Raster `deftm` kernels compiled to bytecode + a **Vulkan** renderer
- **Browser** → the same kernels compiled to **WebAssembly** + a **WebGPU** renderer

It is a showcase for Raster's programming model: the performance-critical numerical
work (terrain gen, physics, meshing math) is written once with `deftm` and compiled
to native code on each platform; the game logic, mesher and renderer are shared
cross-platform Clojure(Script).

Features: biome-aware procedural terrain (11 biomes, per-biome height), caves, ore
veins (coal/iron/gold/diamond), lakes (animated Fresnel water), trees by biome
(oak / birch / spruce), skylight + ambient occlusion + a glowstone block-light
channel (BFS flood, relit on edits), day/night sky with sun, moon and rotating
stars, distance fog, infinite chunk streaming (render ring around the player),
passive mobs (cow/pig/chicken/sheep), block mining/placing, a health/hotbar HUD
(half-hearts), and a fly mode. (The legacy JVM-only survival game — inventory,
crafting, farming, hostile mobs, hunger — lived here previously and remains in git
history.)

## Running

**JVM (Vulkan):**
```bash
clojure -M:vulkan -m valley.vk-walk
```

**Browser (WebGPU — Chrome/Edge; Firefox needs a flag):**
```bash
cd ../asteroids/web
npx shadow-cljs watch cvalley-walk
# open http://localhost:8080/valley-walk.html
```
The wasm kernel module is generated by `gen.clj`:
```bash
clojure -Sdeps '{:paths ["src" "examples"]}' -M examples/valley/gen.clj
```

## Architecture

**Typed numerical core (`deftm`).** `core.clj` holds the kernels (terrain height,
biome, caves, AABB physics + 1-block step-up, geometry). The Raster walker
devirtualizes + inlines these into single JVM methods (no boxing); `gen.clj` compiles
the same kernels to one wasm module and generates the cljs shim (`valley.kernels`).

**Unified shell.** `shell.cljc` is the entire streaming runtime (terrain stream,
render ring, mobs, HUD, mining, fly). `vk_walk.clj` and `web_walk.cljs` are thin
entry points that build a renderer and call it; the only platform-divergent call is
`k/upload-world!` (no-op on JVM, copies the active buffer into wasm memory in the
browser).

**Streaming.** Terrain is a pure function of `(seed, x, y, z)` (no global state), so
chunks regenerate identically regardless of load order. `chunk.cljc` holds a moving
"active buffer" (the loaded ring as one flat array; recenter = copy overlap +
gen new edge strips) and per-column meshes in absolute world coords.

**Renderer.** `raster.render` is a small protocol with Vulkan (`raster.render.vulkan`)
and WebGPU (`raster.render.webgpu`) backends; one shader source (`raster.render.shader`)
emits both GLSL and WGSL.

## File overview

| File | Purpose |
|------|---------|
| `core.clj` | Typed `deftm` kernels (terrain, biome, caves, physics, geometry) |
| `kernels.clj` / `gen.clj` | JVM kernel facade / wasm-module + cljs-shim generator |
| `chunk.cljc` | Deterministic gen, active-buffer streaming, per-column mesher, lighting+AO |
| `walk.cljc` | Player physics, raycast mining, fly mode |
| `cam.cljc` | Camera / projection helpers (perspective, look-at, fly-mvp) |
| `swarm.cljc` | Mob SoA columns + batch physics + box-models with walk animation |
| `tex.cljc` | Procedural block + mob-skin + HUD textures (atlas) |
| `sky.cljc` | Day/night sky shader (gradient, sun, rotating stars) |
| `water.cljc` | Animated wave-normal + Fresnel + reflection water shader |
| `ui.cljc` | 2D HUD overlay (crosshair, hearts, hotbar) |
| `shell.cljc` | Shared streaming game runtime (both platforms) |
| `vk_walk.clj` / `web_walk.cljs` | JVM / browser entry points |
